# Chrome Web Store listing update

## Short description

Hide content cards containing names, words, phrases, or patterns you choose. Private, local, and site agnostic.

## Detailed description

hide-em is a personal attention filter for the web. Add names, keywords, phrases, aliases, or optional regular expressions. hide-em scans page text locally and hides matching content cards as they appear.

Use the toolbar popup to see how many items are hidden, reveal them, pause the current tab or hostname temporarily, and add a rule without leaving the current page. The full settings page provides rule search, local rule testing, editing, rule toggles, JSON backup and restore, debug controls, and excluded sites. Exclude a domain when filtering could interfere with an experience such as live chat. Subdomains are included automatically.

Your configuration stays in the current browser profile. hide-em does not require an account, does not use cloud sync, and has no analytics or backend. Use JSON export and import when you want to move your rules to another browser or device.

Key features:

- Quick add from the toolbar
- Live filtering status and current hidden-item count
- Reveal hidden content and pause the current tab
- Temporary 10-minute and browser-session hostname pauses
- Names, keywords, phrases, aliases, and regular expressions
- Searchable and filterable rule management
- Sample-text rule testing before saving
- Whole-word and case-sensitive options
- Domain and subdomain exclusions
- Merge or replace JSON import
- Local verified storage with recovery backup
- Dynamic support for continuously updated pages
- Works without a Chrome sign-in

## Version 1.2 release notes

This release adds live page status, hidden-item counts, reveal-and-pause controls, temporary tab and hostname pauses, blocklist search and filters, and a local rule tester. It preserves the existing permissions, local-only storage, JSON portability, and universal site-agnostic scanner.

## Permission explanation

Storage is used for the user's local rules, settings, exclusions, recovery backup, and temporary pause state held in browser session memory. Access to web pages is required because hide-em is designed to apply user-created text rules on any site and report page status to its popup. Page text, hidden content, and browsing history are not collected or transmitted.
