# Chrome Web Store listing update

## Short description

Hide content cards containing names, words, phrases, or patterns you choose. Private, local, and site agnostic.

## Detailed description

hide-em is a personal attention filter for the web. Add names, keywords, phrases, aliases, or optional regular expressions. hide-em scans page text locally and hides matching content cards as they appear.

Use the toolbar popup to add a rule without leaving the current page. The full settings page provides editing, rule toggles, JSON backup and restore, debug controls, and excluded sites. Exclude a domain when filtering could interfere with an experience such as live chat. Subdomains are included automatically.

Your configuration stays in the current browser profile. hide-em does not require an account, does not use cloud sync, and has no analytics or backend. Use JSON export and import when you want to move your rules to another browser or device.

Key features:

- Quick add from the toolbar
- Names, keywords, phrases, aliases, and regular expressions
- Whole-word and case-sensitive options
- Domain and subdomain exclusions
- Merge or replace JSON import
- Local verified storage with recovery backup
- Dynamic support for continuously updated pages
- Works without a Chrome sign-in

## Version 1.1 release notes

This release improves save reliability, moves configuration to verified local storage, adds quick add and domain exclusions, hardens dynamic-page scanning, and adds responsive layouts for smaller screens.

## Permission explanation

Storage is used for the user's local rules, settings, exclusions, and recovery backup. Access to web pages is required because hide-em is designed to apply user-created text rules on any site. Page text and browsing history are not collected or transmitted.
